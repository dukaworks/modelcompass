## 🎨 写作风格

### 中文技术写作原则

**1. 开头抓住注意力**
```markdown
❌ 不好：
ModelCompass是一个AI模型比价平台...

✅ 好：
你在用GPT-4写代码，但你知道吗？
同样的需求，用DeepSeek-V3可以省80%成本，而且中文理解更好。

这就是我做ModelCompass的原因。
```

**2. 技术+场景结合**
```markdown
❌ 不好：
/pricing接口返回模型的价格数据...

✅ 好：
想知道哪个模型最便宜？
调用`/pricing`接口，3行代码对比所有模型的百万token价格：

```python
import requests
models = requests.get('/api/pricing').json()
cheapest = min(models, key=lambda x: x['input_price'])
print(f"最便宜的是 {cheapest['name']}")
```
```

**3. 多用列表和代码块**
- 步骤用数字列表
- 对比用表格
- 命令用代码块
- 关键概念用加粗

## 📝 文档模板

### 博客文章模板
```markdown
# 标题（痛点+解决方案）

## 开头（抓住注意力）
- 一个具体场景或问题
- 为什么重要

## 正文（分步骤）
### 1. 问题分析
### 2. 解决方案
### 3. 实际操作

## 总结（ actionable ）
- 关键要点
- 下一步行动
- 相关链接
```

### API文档模板
```markdown
## POST /api/endpoint

**功能**: 一句话描述做什么

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 名称 |

**响应示例**:
```json
{
  "success": true,
  "data": {...}
}
```

**错误码**:
| 状态码 | 说明 |
|--------|------|
| 400 | 参数错误 |
```
